1. Release 빌드 모드와 프로젝트 속성
1) pdb 파일
\projectFile\Release or Debug\.pdb:  프로그램 데이터베이스 파일(소스코드와 기계어와 연결 맵핑해주는 파일)
=>Release가 안되는 이유 (소스코드가 달라져서 맵핑이 안된다. - 소스가 최적화되기 때문)
=>Release 빌드 모드에서 소스 최적화 끄는 방법 (프로젝트 속성->C/C++->최적화)
2) 프로젝트 속성 inline 확장
 => 사용자가 inline으로 지정한 함수만  inline코드로 변환
*Release 빌드 모드로 빌드 시 프로젝트 속성에서 디버깅을 할 수 있게 옵션 조정이 필요하다.(최적화 컴파일) 문제 발생 시 디버깅을 위해서.

2. Debug 빌드 모드
1) Debug 빌드 모드에서는 inline 키워드가 사용되지 않음(프로젝트 속성에서 변경 가능)

 2) big: 낮은 값은 높은 주소에 큰 값은 낮은 주소에
